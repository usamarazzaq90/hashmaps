Basic Terminology

What is a
Hashmap?

Hashing transforms a large, variable-size input into a smaller, fixed-size output (index) using mathematical formulas called hash functions.

01
Highly EfficientSupports search, insert, and delete in O(1) time on average.
02
Hash SetCollection of unique keys (e.g., Python Set, C++ unordered_set).
03
Hash MapKey-value pairs with unique keys (e.g., Python Dict, C++ unordered_map).

ContextWhy Hashing?
Solving inefficiency

Need for
Hash Data Structure

Data is growing exponentially. While Arrays allow storing in O(1) time, searching takes at least O(log n) time. For massive datasets, this becomes highly inefficient.


Hashing bridges this gap by providing a data structure that enables storing, searching, and retrieving data in constant O(1) time.

What is a
Hash Function?

A mathematical algorithm that converts a key (like a word or ID) into a fixed-size numerical index.

Properties of a good Hash Function:

01
EfficiencyFast computation.
02
Uniform DistributionKeys should be spread evenly across the table.
03
Collision MinimizationAvoid mapping multiple keys to the same index.
04
Low Load FactorMaintain optimal capacity.

Why use it?

Advantages of
Hashing

  • Key-Value Support: Ideal for implementing associative data structures.
  • Fast Retrieval: Constant-time O(1) access complexity.
  • High Efficiency: Extremely fast insertion, deletion, and searching.
  • Memory Reduction: Allocates fixed, manageable space.
  • Scalability: Performs exceptionally well with massive datasets.
  • Security: Essential for cryptography and integrity verification.

Edge CasesProblem
When things overlap

Collision in
Hashing

A collision occurs when two or more distinct keys generate the exact same hash value.


Example: Using the sum-of-characters hash function, the strings {"ab", "ba"} both evaluate to 3. They map to the same index, causing a collision which disrupts insertion and searching.


Resolution: Managed using Collision Resolution Techniques (e.g., Chaining or Open Addressing).

MetricsPerformance
Measuring fullness

Load Factor
in Hashing

The ratio that indicates how full a hash table is. It's a decisive parameter for resizing (rehashing) the table.

Load Factor = (Total elements in hash table) / (Size of hash table)

A lower load factor helps determine the efficiency of the hash function and ensures keys are distributed uniformly.
